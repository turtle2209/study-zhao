# RoboMaster 哨兵机器人代码学习规划

## 项目概述

本项目是基于 **STM32F407** 的 RoboMaster 机甲大师哨兵机器人嵌入式控制代码，采用 **FreeRTOS** 实时操作系统，实现了底盘控制、云台控制、发射机构控制、姿态解算等核心功能。

---

## 项目架构总览

```
┌─────────────────────────────────────────────────────────┐
│                    Task 任务层                            │
│  (chassis_task / gimbal_task / shoot_task / INS_task)    │
├─────────────────────────────────────────────────────────┤
│                 Application 应用层                        │
│  (CAN_receive / remote_control / referee / behaviour)   │

├─────────────────────────────────────────────────────────┤
│                 Drivers 驱动层                            │
│  (STM32 HAL库 / CMSIS)                                  │
└─────────────────────────────────────────────────────────┘
```

### 第二阶段：系统启动流程（第2周）

**目标：** 理解整个系统是如何启动的

**阅读顺序：**
1. `Src/main.c` → 系统入口，外设初始化流程
2. `Src/freertos.c` → FreeRTOS任务创建
3. `Hardware/Src/main.c` → 各任务线程的创建和启动

**核心知识点：**
- HAL_Init() → 系统时钟配置 → 外设初始化 → RTOS启动
- STM32时钟树：HSE 12MHz → PLL → SYSCLK 168MHz
- FreeRTOS调度器启动后，main函数中的while(1)永远不会执行

### 第四阶段：PID控制算法（第3-4周）⭐重点

**目标：** 深刻理解PID控制器的实现和调参方法

**文件：** `components/controller/pid.c/h`

#### PID 基本公式

```
输出 = Kp × error + Ki × ∫error + Kd × d(error)/dt
```

#### 本项目PID结构体

```c
typedef struct {
    float Kp, Ki, Kd;      // 三个参数
    float max_out;          // 最大输出限幅
    float max_iout;         // 积分限幅（防积分饱和）
    float error[3];         // 误差历史 [当前, 上次, 上上次]
    float Dbuf[3];          // 微分项历史
    float Pout, Iout, Dout; // 各项输出
    float out;              // 总输出
} pid_type_def;
```

#### 项目中PID的应用场景

| 控制对象 | PID类型 | 环数 | 说明 |
|---------|---------|------|------|
| 底盘电机 | 速度环 | 单环 | 控制轮子转速 |
| 底盘跟随 | 角度环 | 单环 | 底盘跟随云台旋转 |
| 云台电机 | 角度环+速度环 | 双环串级 | 外环角度，内环角速度 |
| 摩擦轮 | 速度环 | 单环 | 控制弹丸发射速度 |
| 拨弹电机 | 速度环 | 单环 | 控制拨弹转速 |
| IMU温控 | 温度环 | 单环 | 维持IMU恒温 |

**串级PID原理（云台为例）：**
```
设定角度 → [角度PID] → 角速度设定值 → [角速度PID] → 电机电流
                ↑                          ↑
           陀螺仪角度                  陀螺仪角速度
```

**调参技巧：**
1. 先调内环（速度环），再调外环（角度环）
2. 先加 Kp 直到震荡，再减小到0.6~0.8倍
3. 加 Ki 消除稳态误差，注意积分限幅
4. 加 Kd 抑制超调，但不宜过大（会放大噪声）

### 第五阶段：底盘控制（第4-5周）

**目标：** 理解麦克纳姆轮运动学和底盘控制逻辑

**文件：** `Task/chassis_task.c/h` + `application/chassis_behaviour.c/h`

#### 5.1 麦克纳姆轮运动学

**轮子布局（俯视图）：**
```
    1 ── 前 ── 2
    |           |
    |     ×     |    x轴：前进方向
    |           |    y轴：左方向
    4 ── 后 ── 3    wz：逆时针旋转
```

**正运动学（底盘速度 → 轮子转速）：**
```c
wheel[0] = R * (-√2/2 * vx - √2/2 * vy + L * wz)
wheel[1] = R * (-√2/2 * vx + √2/2 * vy + L * wz)
wheel[2] = R * ( √2/2 * vx + √2/2 * vy + L * wz)
wheel[3] = R * ( √2/2 * vx - √2/2 * vy + L * wz)
```

#### 5.2 底盘控制模式

| 模式 | 说明 | 应用场景 |
|------|------|---------|
| CHASSIS_ZERO_FORCE | 无力模式 | 电机不输出 |
| CHASSIS_NO_MOVE | 保持不动 | 底盘锁定 |
| CHASSIS_FOLLOW_GIMBAL | 跟随云台 | 正常行进 |
| CHASSIS_TOP | 小陀螺 | 规避弹丸 |
| CHASSIS_NO_FOLLOW_GIMBAL | 不跟随 | 特殊操作 |

#### 5.3 底盘跟随云台

核心思想：底盘通过PID控制旋转角速度wz，使云台相对角度趋近于0。

### 第六阶段：云台控制（第5-6周）

**目标：** 理解云台双环PID控制和多种控制模式

**文件：** `Task/gimbal_task.c/h` + `application/gimbal_behaviour.c/h`

#### 6.1 控制框架

```
遥控器/鼠标/自瞄 → 角度设定值
                     ↓
              [角度环PID] ← 陀螺仪角度反馈
                     ↓
              角速度设定值
                     ↓
              [速度环PID] ← 陀螺仪角速度反馈
                     ↓
              电机电流 → CAN发送
```

#### 6.2 关键概念

- **绝对角度：** 世界坐标系下的角度（来自陀螺仪）
- **相对角度：** 相对底盘的角度（来自电机编码器）
- **角度限位：** 防止云台转到机械极限（Pitch: -0.25~0.7 rad）
- **角度格式化：** 处理角度跨越±π的连续性问题

#### 6.3 自瞄模式
- 接收上位机发送的目标角度偏差
- 叠加到当前角度设定值上
- 使用专门的自瞄PID参数（与手动模式不同）

---

### 第七阶段：发射机构控制（第6周）

**目标：** 理解摩擦轮和拨弹机构的控制逻辑

**文件：** `Task/shoot_task.c/h`

#### 7.1 发射状态机

```
SHOOT_STOP → SHOOT_READY → SHOOT_BULLET → SHOOT_CONTINUE_BULLET
  (关闭)      (开摩擦轮)    (单发拨弹)      (连发拨弹)
```

#### 7.2 关键机制

- **堵转检测：** 拨弹电机转速低于阈值持续一段时间 → 反转一段时间
- **热量管理：** 从裁判系统获取实时热量，超热量自动停止
- **摩擦轮缓启动：** 启动时限制最大电流，防止电流冲击
- **射速控制：** 根据机器人等级调整摩擦轮转速

### 第八阶段：IMU姿态解算（第7周）⭐进阶

**目标：** 理解AHRS算法和IMU数据融合

**文件：** `Task/INS_task.c/h` + `components/algorithm/AHRS.c/h`

#### 8.1 传感器

| 传感器 | 型号 | 提供数据 | 接口 |
|--------|------|---------|------|
| 加速度计 | BMI088 | ax, ay, az | SPI+DMA |
| 陀螺仪 | BMI088 | wx, wy, wz | SPI+DMA |
| 磁力计 | IST8310 | mx, my, mz | I2C |

#### 8.2 AHRS算法流程

```
传感器原始数据 → 校准补偿 → 低通滤波 → 四元数更新 → 欧拉角转换
```

#### 8.3 关键输出

- `INS_data.angle_yaw` — 偏航角（用于云台Yaw控制）
- `INS_data.angle_pitch` — 俯仰角（用于云台Pitch控制）
- `INS_data.wx/wy/wz` — 角速度（用于PID内环）

## 学习技巧与注意事项

### 代码阅读技巧

1. **从Task入手：** 每个Task文件就是一个独立的控制流程，从`xxx_task()`入口函数开始读
2. **先读.h再读.c：** 头文件包含结构体定义、宏定义、函数声明，能快速了解模块功能
3. **关注数据流：** 追踪数据从传感器读取 → 处理 → 输出到执行器的完整链路
4. **画框图：** 自己手画每个模块的控制框图，加深理解
5. **对照硬件手册：** 阅读RoboMaster C板原理图和电机手册

### 调试技巧

1. **使用Keil Watch窗口：** 实时观察变量值变化
2. **J-Link RTT或串口打印：** 输出关键变量用于分析
3. **逐步使能：** 先让一个电机转，再加其他功能
4. **记录参数：** 每次调参记录PID值和效果

### 常见坑点

1. **CAN总线ID冲突：** 同一CAN线上不同电机的发送帧ID不能相同
2. **编码器溢出：** 电机编码器是0~8191循环，需要处理跨零点
3. **角度连续性：** yaw角会跨越±π，需要用`rad_format()`格式化
4. **任务优先级：** 高频任务（IMU 1ms）优先级要高于低频任务
5. **指针安全：** 使用指针前一定要判空（本项目中有大量`if (xxx == NULL) return;`）
6. **热量超限：** 连续射击前必须检查热量，超限会被判罚

### 项目编码规范（从代码中观察到的）

- 任务函数统一格式：`void xxx_task(void const *pvParameters)`
- 变量命名使用下划线分隔：`chassis_move_t`
- 宏定义全大写：`CHASSIS_MOTOR_SPEED_KP`
- 静态函数在文件头部声明
- 使用`extern`暴露接口给其他模块
